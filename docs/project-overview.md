## Purpose

**QuantVista** is a multi-tenant SaaS that analyzes the **Nifty 200** universe and produces
institutional-style research: multi-factor **scores** (fundamental, momentum, quality, sentiment,
risk), a composite **research signal**, **portfolio construction & optimization**, **risk analytics**,
**news/sentiment** intelligence, and **bias-controlled backtesting**. Positioned as a **research &
analytics tool, not a registered investment advisor** (locked decision D1).

## Architecture at a glance

- **Modular monolith**: bounded contexts (`identity`, `market_data`, `analytics`, `portfolio`,
  `news`, `alerts`, `platform/core`) communicate **only** through published Python `Protocol`/ABC
  interfaces or Redis Streams domain events. `import-linter` enforces the dependency DAG.
- **Same image, three roles**: one backend image runs `api` (uvicorn), `worker` (Celery), `beat`
  (scheduler) by command.
- **Two data domains**: global reference/market data (no `tenant_id`, no RLS, written by jobs) vs.
  tenant data (`tenant_id` + Postgres RLS, written by user actions).
