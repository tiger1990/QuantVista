# QuantVista — Database (Alembic migrations)

Production-grade DDL for the schema in [`../plans/03-data-architecture.md`](../plans/03-data-architecture.md).
Migrations are **hand-written** (not autogenerated) because the schema relies on three things SQLAlchemy
autogenerate does not model well: **declarative range partitioning**, **bitemporal point-in-time columns**, and
**Row-Level Security (RLS)**.

## Design rules encoded here

- **Two data domains** (`../plans/02` §1, `../plans/03` §2):
  - **Global** reference/market tables — no `tenant_id`, no RLS, written only by jobs.
  - **Tenant** tables — `tenant_id UUID NOT NULL`, **RLS `ENABLE` + `FORCE`**, isolated via
    `app_current_tenant()`.
- **Tenant isolation by construction (D6):** every request sets `SET LOCAL app.tenant_id = '<uuid>'` inside
  its transaction; RLS policies do the rest. The app must connect as a **non-superuser without `BYPASSRLS`**.
- **Point-in-time correctness (`../plans/03` §5):** `fundamentals` is bitemporal
  (`period_end` + `knowledge_from/knowledge_to`); revisions insert a new version and close the prior one.
- **Survivorship-free universe:** `stocks.delisted_on` + `index_constituents` (PIT membership).
- **Partitioning:** `daily_prices`, `technical_indicators`, `factor_values`, `scores` are `PARTITION BY RANGE
  (date)` monthly; a `create_month_partition()` helper + a maintenance job (or pg_partman) add partitions.
- **Money/ratios are `NUMERIC`, never float.** Timestamps `TIMESTAMPTZ` in UTC.

## Usage

```bash
export DATABASE_URL=postgresql+psycopg://quantvista:***@localhost:5432/quantvista
cd db
alembic upgrade head           # apply all migrations
alembic downgrade -1           # roll back one
alembic current                # show current revision
psql "$DATABASE_URL" -f seeds/seed_reference.sql   # load reference/seed data
```

### Operational notes
- **Migration safety (`../plans/03` §9):** forward-only in prod; use expand → backfill → contract for
  zero-downtime column/constraint changes. Never destructive in a single release.
- **RLS tests are CI-gated** (`../plans/07` §3, `../plans/sprints` QV-004/061): a cross-tenant access attempt
  must return zero rows / be rejected.
- **Partition maintenance:** schedule `SELECT create_month_partition('daily_prices', date_trunc('month',
  now() + interval '1 month')::date);` (and for the other partitioned tables) ahead of each month.
